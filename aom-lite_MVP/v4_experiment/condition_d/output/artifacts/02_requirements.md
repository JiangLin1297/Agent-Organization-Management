## 产品需求文档

### 1. 产品定位（48字）
面向休闲网页用户的经典贪吃蛇游戏，追求现代美观的UI、流畅的操控体验，适配桌面与移动设备。

### 2. 功能需求（198字）
| 功能 | 优先级 | 描述 |
|------|--------|------|
| 三阶段界面 | P0 | 实现开始、游戏、结束三个独立界面。 |
| 键盘操控 | P0 | 支持方向键控制蛇的移动方向，响应需即时。 |
| 计分系统 | P0 | 每吃一个食物+10分，实时显示。 |
| 游戏规则 | P0 | 蛇撞墙或撞自身则游戏结束。 |
| 重新开始 | P0 | 游戏结束界面提供按钮，返回开始界面。 |
| 响应式布局 | P1 | 界面元素与游戏画布自适应屏幕尺寸。 |
| 触摸操控 | P1 | 移动端支持滑动屏幕控制方向。 |
| 动态背景 | P2 | 开始与结束界面有柔和的渐变或粒子动画。 |
| 得分动画 | P2 | 计分增加时有缩放或弹跳动效。 |

### 3. 界面设计（197字）
- **开始界面**：居中布局，包含游戏标题（如“Neon Snake”）、简要游戏说明（键盘/滑动）、以及醒目的“开始游戏”按钮。
- **游戏界面**：顶部为计分板（显示分数），中央为正方形游戏画布，角落可添加简易的方向键示意（移动端重要）。
- **结束界面**：居中卡片式布局，显示“游戏结束”、最终得分、历史最高分（本地存储）、以及“重新开始”按钮。
- **配色方案**：主色采用深蓝色（#0a192f）背景，营造科技感。蛇身使用明亮的青绿色（#64ffda），食物为亮黄色（#ffd700），文字与边框使用浅灰色（#ccd6f6），确保高对比度与可读性。

### 4. 交互规范（148字）
- **操作流程**：点击“开始” -> 使用方向键/滑动控制蛇移动 -> 蛇死亡自动跳转结束界面 -> 点击“重新开始”循环。
- **反馈机制**：得分时分数数字动态变化；蛇碰撞瞬间，蛇身或障碍物有短暂高亮闪烁。
- **异常处理**：游戏开始时，忽略无效的方向输入（如反向移动）；窗口失焦时暂停游戏循环，避免计时错乱。

### 5. 验收标准（95字）
- [ ] 三个界面可按流程正常切换，无卡顿。
- [ ] 方向键与触摸滑动操控流畅，蛇移动速度适中且稳定。
- [ ] 计分准确，撞墙/撞自身后游戏正确结束。
- [ ] 界面布局美观，适配不同桌面与移动屏幕尺寸。
- [ ] 代码为单个HTML文件，可直接在浏览器打开运行。
