# 3.X Theoretical Propositions

This section formalizes the core theoretical claims of this paper as two propositions. We do not claim formal mathematical proofs; rather, we present logically rigorous arguments with explicitly stated assumptions, supported by empirical evidence from our experiments.

## Proposition 1: Context Accumulation Effect (CAE)

**Statement.** Let M be an LLM agent performing iterative self-improvement over n rounds, where in each round i the agent receives as input the complete output of round i−1 plus a refinement instruction. Let t(i) denote the token count of the output at round i. Under the assumption that output size does not decrease across rounds (t(i) ≥ t(i−1) for all i), the total token consumption T(n) satisfies:

> **T(n) = Ω(n²)**

**Assumptions.**
- (a) Each round requires the full output of the previous round as input context (no compression or summarization).
- (b) Output size is non-decreasing: each refinement preserves or expands the artifact.
- (c) The refinement instruction has non-trivial length (constant lower bound).

**Argument.** The input to round i consists of the refinement instruction (constant size c) plus the full output of round i−1. Under assumption (b), t(i) ≥ t(1) for all i ≥ 1. The total token consumption is:

T(n) = Σᵢ₌₁ⁿ [input_tokens(i) + output_tokens(i)]

where input_tokens(i) ≥ c + t(i−1) ≥ c + t(1). Since each round processes at least the full output of the previous round, and output sizes are non-decreasing, the cumulative cost grows at least as the sum of an arithmetic sequence, yielding T(n) = Ω(n²).

**Empirical validation.** In our experiments, Condition B (iterative 7-round refinement) consumed 90,437 tokens (DeepSeek) and 102,417 tokens (MiMo), compared to 6,000 and 8,251 tokens for single-pass generation (Condition A). The ratio B/A ≈ 15x for 7 rounds is consistent with quadratic growth (7² = 49, though the constant factor and early-round dynamics produce a lower observed ratio).

**Boundary conditions.** Proposition 1 holds when:
- (a) The agent has no mechanism to compress or summarize prior context.
- (b) The task requires preserving the full artifact (e.g., code generation, where partial output is non-functional).

When condition (a) is violated — e.g., through context compression — the growth rate may be reduced but remains superlinear unless compression is lossless.

## Proposition 2: Computational Decomposition Effect (CDE)

**Statement.** Let M be a set of k specialized LLM agents organized in a pipeline, where each agent i receives as input only the output of its direct predecessor(s) plus a task-specific instruction. Let t(i) denote the output size of agent i. Under the assumption that each agent's output size is bounded by a constant B (t(i) ≤ B for all i), the total token consumption T(k) satisfies:

> **T(k) = O(k · B)**

That is, the total cost grows **linearly** in the number of pipeline stages, with the growth rate determined by the per-stage output bound B — not by the cumulative history.

**Assumptions.**
- (a) Each agent receives only its direct dependencies (not the full pipeline history).
- (b) Each agent's output size is bounded: t(i) ≤ B for some constant B independent of k.
- (c) The pipeline has no feedback loops (each agent executes exactly once).

**Argument.** The input to agent i consists of a task instruction (constant size) plus the outputs of its direct predecessors (bounded by B per predecessor). If each agent has at most d predecessors (constant fan-in), then input_tokens(i) = O(d · B) = O(B). Summing over k agents:

T(k) = Σᵢ₌₁ᵏ [input_tokens(i) + output_tokens(i)] = Σᵢ₌₁ᵏ O(B) = O(k · B)

This is linear in k, in contrast to the quadratic growth of Proposition 1.

**Empirical validation.** Condition C (OIMAC with Context Controller, k=7 roles) consumed 45,581 tokens (DeepSeek) and 48,468 tokens (MiMo), compared to 90,437 and 102,417 for iterative refinement (Condition B). The C/B ratio of ~2.0x represents the difference between linear and (approximately) quadratic growth for n=7 rounds. The decomposition into CDE (~70%) and Context Controller (~30%) further confirms that pipeline structure is the primary efficiency driver.

**Relationship to CAE.** Proposition 2 shows that the pipeline architecture of OIMAC converts the quadratic growth of CAE (Proposition 1) into linear growth. The key mechanism is structural: by ensuring that no agent ever processes the cumulative output of all preceding agents, the pipeline eliminates the superlinear dependency chain that CAE exploits.

**Boundary conditions.** Proposition 2 holds when:
- (a) The task can be decomposed into k independent or semi-independent stages.
- (b) Each stage's output is bounded (does not grow with pipeline depth).
- (c) The pipeline is acyclic (no iterative feedback).

When condition (c) is violated — e.g., when a testing stage feeds back to an implementation stage — the architecture degenerates toward iterative refinement, and CAE re-emerges.

## Proposition 3: Real-World Baseline Gap

**Statement.** Multi-agent systems that use global shared context (where all agents observe all prior outputs) — the default architecture of frameworks such as AutoGen and CrewAI — exhibit token consumption that is:

> **C_shared > C_pipeline**

where C_shared is the cost of the shared-context architecture and C_pipeline is the cost of the pipeline architecture (OIMAC without Context Controller).

**Argument.** In a shared-context architecture, each agent i receives the full output of all i−1 preceding agents. The total context processed is:

C_shared = Σᵢ₌₁ᵏ [Σⱼ₌₁ⁱ⁻¹ t(j) + t(i)]

which grows as O(k² · B) — the same quadratic pattern as CAE, but distributed across multiple agents rather than multiple rounds. In contrast, the pipeline architecture (Proposition 2) processes O(k · B) total context.

**Empirical validation.** Condition E (shared-context baseline) consumed 64,634 tokens (DeepSeek), compared to 60,894 for Condition D (pipeline without CC) and 45,581 for Condition C (pipeline with CC). The E > D ordering confirms that even without Context Controller, pipeline structure outperforms shared-context architecture. The C/E ratio of ~1.4x demonstrates the additional benefit of combining pipeline structure with context control.
