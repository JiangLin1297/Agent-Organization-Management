# 3.3 Theoretical Propositions

This section formalizes the core theoretical claims of this paper as empirical scaling propositions. We do not claim formal mathematical lower bounds; rather, we identify consistent scaling patterns observed across multiple experimental conditions, models, and tasks, and provide intuitive arguments for why these patterns arise.

## Proposition 1: Context Accumulation Effect (CAE)

**Statement.** When an LLM agent iteratively refines its own output over n rounds — where each round receives the complete output of the previous round as input context — total token consumption T(n) exhibits **quadratic-like growth** as a function of the number of refinement rounds.

**Assumptions.**
- (a) Each round requires the full output of the previous round as input context (no compression or summarization).
- (b) Output size is non-decreasing: each refinement preserves or expands the artifact.
- (c) The refinement instruction has non-trivial length (constant lower bound).

**Argument.** The input to round i consists of the refinement instruction (constant size c) plus the full output of round i−1. Under assumption (b), the output size at round i is at least as large as at round 1. Since each round processes at least the full output of the previous round, and output sizes are non-decreasing, the cumulative cost grows at least as the sum of an arithmetic sequence. This produces a scaling pattern consistent with quadratic growth, though the exact exponent depends on task-specific factors such as output growth rate and round-to-round variance.

**Empirical validation.** Condition B (iterative 7-round refinement) consumed 90,437 tokens (DeepSeek) and 102,417 tokens (MiMo), compared to 6,000 and 8,251 tokens for single-pass generation (Condition A). The observed B/A ratio of approximately 15x for 7 rounds is consistent with superlinear growth. While we do not claim a precise exponent, the pattern is robust across two architecturally distinct models and two heterogeneous tasks.

**Important caveat.** We do not claim a formal Ω(n²) lower bound, as the exact growth rate depends on factors we do not control (e.g., output size variance, model-specific tokenization). What we demonstrate empirically is that (1) iterative refinement incurs substantially higher cost than single-pass generation, and (2) the cost growth is superlinear in the number of rounds. The quadratic characterization is a descriptive model, not a proven bound.

**Boundary conditions.** The CAE pattern is most pronounced when:
- (a) The agent has no mechanism to compress or summarize prior context.
- (b) The task requires preserving the full artifact (e.g., code generation, where partial output is non-functional).

When condition (a) is violated — e.g., through context compression — the growth rate may be reduced but remains superlinear unless compression is lossless.

## Proposition 2: Computational Decomposition Effect (CDE)

**Statement.** When a task is decomposed into a pipeline of k specialized LLM agents — where each agent receives only the output of its direct predecessor(s) rather than the full pipeline history — total token consumption T(k) grows **approximately linearly** in the number of pipeline stages, with the growth rate determined by the per-stage output size rather than the cumulative history.

**Assumptions.**
- (a) Each agent receives only its direct dependencies (not the full pipeline history).
- (b) Each agent's output size is bounded and does not grow with pipeline depth.
- (c) The pipeline has no feedback loops (each agent executes exactly once).

**Argument.** The input to agent i consists of a task instruction (constant size) plus the outputs of its direct predecessors (bounded per predecessor). If each agent has a bounded number of predecessors, then the input to each agent is bounded independently of the total number of pipeline stages. Summing over k agents yields total cost that scales linearly with k, in contrast to the superlinear growth observed under CAE.

**Empirical validation.** Condition C (OIMAC with Context Controller, k=7 roles) consumed 45,581 tokens (DeepSeek) and 48,468 tokens (MiMo), compared to 90,437 and 102,417 for iterative refinement (Condition B). The C/B ratio of approximately 2.0x represents the practical difference between linear and superlinear scaling for 7 stages. The mechanism decomposition (CDE ~70%, Context Controller ~30%) further confirms that pipeline structure is the primary efficiency driver.

**Relationship to CAE.** The pipeline architecture of OIMAC converts the superlinear growth of CAE into approximately linear growth. The key mechanism is structural: by ensuring that no agent ever processes the cumulative output of all preceding agents, the pipeline eliminates the dependency chain that CAE exploits.

**Boundary conditions.** The CDE pattern holds when:
- (a) The task can be decomposed into k independent or semi-independent stages.
- (b) Each stage's output is bounded (does not grow with pipeline depth).
- (c) The pipeline is acyclic (no iterative feedback).

When condition (c) is violated — e.g., when a testing stage feeds back to an implementation stage — the architecture degenerates toward iterative refinement, and CAE re-emerges.

## Proposition 3: Real-World Baseline Gap

**Statement.** Multi-agent systems that use global shared context — where all agents observe all prior outputs, the default architecture of frameworks such as AutoGen and CrewAI — exhibit token consumption that exceeds that of pipeline-based architectures with bounded context passing.

**Argument.** In a shared-context architecture, each agent i receives the full output of all i−1 preceding agents. The total context processed grows with the square of the number of agents, since each agent must process the cumulative history. In contrast, the pipeline architecture ensures that each agent processes only its direct dependencies, yielding linear growth.

**Empirical validation.** Condition E (shared-context baseline) consumed 62,459 tokens (DeepSeek, N=5, std=1,745), compared to 60,894 for Condition D (pipeline without CC) and 45,581 for Condition C (pipeline with CC). The E > D ordering confirms that even without Context Controller, pipeline structure outperforms shared-context architecture. The C/E ratio of approximately 1.4x (p < 0.001) demonstrates the additional benefit of combining pipeline structure with context control.

**Practical implication.** OIMAC outperforms not only naive iteration but also widely-used shared-context architectures. This suggests that the default design of many multi-agent frameworks — shared conversation threads where all agents observe all prior outputs — is structurally suboptimal for token efficiency.
