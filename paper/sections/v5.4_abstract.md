# Abstract (V5.4 — Problem-Driven Edition)

## Problem

Multi-agent systems built on large language models (LLMs) suffer from a structural cost pathology we term the **Context Accumulation Effect (CAE)**: when an LLM agent iteratively refines its own output, each round must embed the complete artifact from the previous round, causing token consumption to grow at least quadratically — O(n²) in the number of refinement rounds. This is not an implementation inefficiency but a fundamental property of iterative self-improvement architectures. As organizations deploy increasingly complex multi-agent workflows, CAE imposes a hard economic ceiling on scalability.

## Insight

We observe that the root cause of CAE is not "too much context" but **incorrect computational structure**. Existing approaches — prompt optimization, context compression, retrieval-augmented generation — treat symptoms by reducing the volume of context. We argue instead that the problem lies in the *topology* of computation: iterative self-improvement chains all computation through a single bottleneck, forcing each step to re-process the entire history. Restructuring the computation into a pipeline of specialized roles, where each role receives only its direct dependencies, eliminates the quadratic growth at its source.

## Method

We introduce **OIMAC** (Organizational Intelligence Multi-Agent Coordination), a structural solution that decomposes monolithic iterative refinement into a multi-role pipeline. OIMAC instantiates organizational design principles — division of work, unity of command, scalar chain — as executable coordination protocols for LLM agents. A key component is the **Context Controller**, which enforces bounded context passing: each role receives only the artifacts it directly depends on, with context size capped at a configurable threshold. OIMAC is not an optimization trick layered onto existing architectures; it is a fundamentally different computational topology.

## Results

We validate OIMAC through 66 independent experimental runs across two architecturally distinct LLMs (DeepSeek Chat, MiMo v2.5 Pro), two heterogeneous tasks (perler bead pattern generation, data analysis report), and five experimental conditions including a real-world baseline simulating AutoGen/CrewAI-style multi-agent systems. Key findings:

- **~2.0x token reduction** versus iterative refinement (C/B ratio, p < 0.001, Cohen's d > 5.6)
- **~1.4x token reduction** versus real-world multi-agent baselines (C/E ratio)
- **Cross-model consistency**: results replicate across DeepSeek and MiMo
- **Cross-task consistency**: results replicate across code-generation and data-analysis tasks
- **Quality preservation**: all outputs pass automated quality checks (20–24/25)
- **Mechanism decomposition**: ~70% of gains from pipeline decomposition (CDE), ~30% from Context Controller

These results demonstrate that **organizational structure is a first-class computational primitive** — not a metaphor, but a measurable, reproducible source of efficiency in LLM-based multi-agent systems.
