## 产品需求文档

### 1. 产品定位（≤50字）
一款面向休闲玩家的、视觉与操控现代化的经典贪吃蛇网页游戏，提供流畅的游玩体验与美观的界面。

### 2. 功能需求（≤200字）
| 功能 | 优先级 | 描述 |
|------|--------|------|
| 游戏核心循环 | P0 | 蛇的移动、食物生成、碰撞检测（墙壁、自身）、得分计算。 |
| 三种操控方式 | P0 | 支持键盘方向键（桌面）、屏幕虚拟方向键（移动端）及触屏滑动操控。 |
| 界面流程 | P0 | 包含清晰的开始、游戏进行中、游戏结束三个独立界面状态。 |
| 计分与速度系统 | P0 | 实时显示当前分数，蛇随分数增加而逐渐加速。 |
| 响应式布局 | P1 | 界面与游戏区域自动适配桌面与移动端不同屏幕尺寸。 |
| 视觉反馈 | P1 | 蛇吃到食物、游戏结束时有明确的动画与提示效果。 |
| 重新开始功能 | P0 | 游戏结束后提供“重新开始”按钮，可随时重置游戏。 |

### 3. 界面设计（≤200字）
- **开始界面**：居中显示游戏标题（如“Modern Snake”）与显眼的“开始游戏”按钮。背景简洁。
- **游戏界面**：顶部显示“分数: [数值]”。中央为网格化游戏区域，包含蛇与食物。移动端下方显示虚拟方向键。
- **结束界面**：覆盖游戏界面，居中显示“游戏结束”与最终得分，并提供“重新开始”按钮。
- **配色方案**：主背景为深灰色(#1a1a1a)或黑色，游戏区域为深色(#2d2d2d)。蛇身为高饱和度亮色（如#00ff88），食物为对比色（如#ff4757）。UI文字与按钮使用白色或浅灰色。

### 4. 交互规范（≤150字）
- **操作流程**：开始界面点击按钮/按任意键 → 进入游戏 → 通过键盘/触屏控制蛇移动 → 碰撞后进入结束界面 → 点击按钮重新开始。
- **反馈机制**：蛇吃到食物时，分数增加，蛇身短暂闪烁或伸长动画。游戏结束时，界面轻微震动或变暗。
- **异常处理**：当蛇移动到边界时，立即判定游戏结束。防止蛇在反向移动时（例如向右时按左键）导致的自身碰撞误判。

### 5. 验收标准（≤100字）
- [ ] 三个界面（开始、游戏、结束）均可正确显示与切换。
- [ ] 键盘方向键与移动端触屏/虚拟按键操控响应及时、准确。
- [ ] 分数随吃食物正确增加，蛇速随之提升。
- [ ] 碰撞墙壁或自身后，游戏正确结束并显示最终得分。
- [ ] 在主流桌面浏览器及移动端浏览器中，界面布局自适应，可正常游玩。
